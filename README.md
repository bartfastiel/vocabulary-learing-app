It uses native **Web Components** for full encapsulation â€” every component includes its own logic, markup, and styles.

## ğŸ§± Responsibilities by File

### **index.html**
- Minimal root HTML page.
- Loads `<app-shell>` from `core/app-shell.js`.
- Defines no styles or logic itself.

### **core/app-shell.js**
- Root container for the application.
- Mounts `<vocab-trainer>` or `<rocket-game>`.
- Provides top-level switching logic between trainer and game.

### **core/audio.js**
- The **only** shared helper.
- Provides:
```js
  playSound("ding" | "buzz");
  playVoice(wordEnglish);
```

* Handles consistent sound effect and text-to-speech playback.

### **vocab/vocab.js**

* The **central orchestrator** of the learning flow.
* Imports all question and answer components at the top (no dynamic loading).
* Defines:

    * all possible question types
    * all possible answer types
    * all valid questionâ€“answer combinations (modes)
* Randomly selects the next mode and vocabulary entry.
* Creates the corresponding question and answer elements.
* Listens for `answered` events and updates scores through `points.js`.
* Integrates with `core/audio.js` for feedback sounds.

### **vocab/points.js**

* Handles:

    * localStorage for points, streak, and highscore
    * score updates and reset logic
    * streak record tracking
* Exposes simple methods for increment/decrement and retrieval.

### **vocab/question/** components

Each defines one prompt type and encapsulates its visual and logical behavior.

| Component                        | Responsibility                                 |
| -------------------------------- | ---------------------------------------------- |
| `question-wordGerman.js`         | Shows the German word as text.                 |
| `question-wordEnglish.js`        | Shows the English word (for reverse learning). |
| `question-image.js`              | Displays an image as the prompt.               |
| `question-voiceEnglish.js`       | Plays an English voice clip.                   |
| `question-descriptionEnglish.js` | Shows an English descriptive hint.             |

All question components expose the word data via a `word` property and display their prompt immediately upon assignment.

### **vocab/answer/** components

Each defines an input or response method, encapsulated and self-contained.

| Component                             | Responsibility                                   |
| ------------------------------------- | ------------------------------------------------ |
| `answer-chooseWordEnglish.js`         | Multiple-choice with English text answers.       |
| `answer-chooseWordGerman.js`          | Reverse mode: choose German translation.         |
| `answer-chooseImage.js`               | Multiple-choice with images.                     |
| `answer-chooseVoiceEnglish.js`        | Choose between voice clips.                      |
| `answer-typeWordEnglish.js`           | Text input to type the English word.             |
| `answer-fillMissingLettersEnglish.js` | Fill missing letters of the English word.        |
| `answer-speakWordEnglish.js`          | Speak the English word aloud (future extension). |

Each emits an `answered` event:

```js
this.dispatchEvent(new CustomEvent("answered", {
  bubbles: true,
  detail: { correct: true/false }
}));
```

### **game/** domain

Self-contained mini-game unrelated to vocab logic.

* `rocket-game.js` â€” main game loop, controls, and rendering.
* `rocket-hud.js` â€” HUD with combo bars, counters, and coin stats.

## ğŸ§  Naming Conventions

### Mode structure

```

```

* `questionFormat` = how the vocabulary is presented (e.g., `wordGerman`, `image`, `voiceEnglish`)
* `answerFormat` = how the learner responds (e.g., `chooseWordEnglish`, `typeWordEnglish`)
